HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • The head and body are semantic elements
  • Declarations are not case sensitive but elements are
  • Only use the h1 heading once
  • p element is for paragraphs
  • li element is for lists
  • ul element is for unordered lists
  • img element is for images, which will include src, alt, width, and height attributes
  • br element is for line breaks
  • a element is for links with href attribute for the link itself followed by a description before closing the /a
  • section elements can have id and class attributes
  • div and span are non-semantic elements used to create a division and create an inline division respectively

CSS

  • A margin indicates how much space we want around the outside of an element
  • A border separates the margin from the padding
  • A padding indicates how much space we want around the content inside an element
  • Content, padding, border, margin are components of teh CSS box model
  • CSS content is structured with the selector followed by the declaration block
  • CSS code can be inside the HTML file impacting a specific element, as inline CSS
  • CSS code can be inside the head of the HTML file, as an internal file sheet impacting the whole page
  • Most commonly, CSS code is in an external style sheet, linked to the HTML file in the head
  • External file sheets follow the Separation of Concerns, with code for different uses in different files
  • A period . designates a class selector
  • An asterisk * selects all elements
  • A pound sign # selects an element

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A submits the files in the working branch to the staging area of Git
